Ctraj project

EPIC —> Ensure CTraj conforms to best practices for open source software development, is in a state where users can contribute code, and is distributed both practically (conda) and intellectually (publication)




User story 0: “Make smFRET functionality output custom sized histogram"
Final changes to COCOFRET for formatting 


User story 1: "I want to use CTraj with Python 3”

Update CTraj codebase to be compatible with Python 3 while maintaining Python 2 backwards compatibility 
Test using existing ’testing framework’ 
Make sure code is ‘clean’ (conforms to PEP8, docstrings in numpy format, ensure zero code redundancy, clean up COCOFRET functions to only have one!)


User story 2: "I want good documentation for CTraj”

Create a new cookie cuter template and integrate CTraj into that template to ‘hook in’ all the goodness of cookie cutter 
Update docstrings on all CTraj files using numpy-style docstrings 
Get Sphinx working to autogenerate all HTML documentation
Get people in lab to read and ‘check’ documentation to make sure it makes sense, is readable
Include documentation on best practices for ‘adding a new analysis function'


User story 3: "I want to be confident CTraj works and to be able to test my new function doesn’t break everything!”

Build a complete integrative testing pipeline using Pytest 
  • NB: Single protein analysis should include
    • Protein only (+all combinations of N/C termini)
    • Protein + ions (+all combinations of N/C termini)
    • Protein + protein (+all combinations of N/C termini)
  • Multi-protein analysis should include
    • Protein + protein (+all combinations of N/C termini)
Achieve > 95% code coverage (pytest-cov)
Ensure CTraj works with trajectories from a range of simulation engines (CAMPARI, NAMD, GROMACS, Amber, CHARMM, OpenMM)
    • Solicit trajectories from many different groups 
    • Build interface classes as needed to fix differences in how different trajectories represent (should not be necessary in principle)

User story 4: “I want CTraj to be fully open source ”

Push CTraj to public Github repo 
Connect documentation to readthedocs
Connect testing framework to travisCI
Ensure both update on a push to the repo 
Integrate badges into GitHub page

User story 5: “I to be able to install CTraj vis conda forge”

Create conda forge recipe and distribute via conda forge 
Ensure this works on OSX and Linux (!)
  •     Get colleagues to double check installation works well (Jared, Davidé, Greg, Lia etc.)

User story 6: “I want to be able to read about CTraj in a cool journal!”

Write up CTraj paper
  • Emphasize new types of analysis implemented in CTraj
